# ECShop 项目介绍

## 1. 项目背景

ECShop 是一套 B2C 电商系统。本项目使用本机部署的 ECShop 演示站作为被测系统，模拟企业测试团队从接收项目到发布质量评估的完整工作过程。

本项目不是单独练习某一个测试工具，而是围绕同一套电商业务，将不同测试技术应用到合适的测试阶段，并保留可以用于复盘和面试展示的过程产物。

## 2. 被测系统

### 2.1 Web 前台

地址：http://localhost/ECShop/

已确认的页面和入口：

| 功能 | 页面地址 |
| --- | --- |
| 商城首页 | `/ECShop/index.php` |
| 登录与用户中心 | `/ECShop/user.php` |
| 用户注册 | `/ECShop/user.php?act=register` |
| 商品搜索 | `/ECShop/search.php` |
| 购物车与结算 | `/ECShop/flow.php` |
| 商品分类 | `/ECShop/category.php?id={分类ID}` |
| 商品详情 | `/ECShop/goods.php?id={商品ID}` |

当前首页实际展示的分类包括：

- 数码
- 联想
- 华硕
- 外星人
- 食物
- 海鲜
- 电器

当前首页可见商品示例：海尔立式空调。

### 2.2 后台管理端

地址：http://localhost/ECShop/admin/

后台页面标题已确认为“ECSHOP 管理中心”。后台将在获得合法测试账号后用于商品、会员和订单管理测试。

### 2.3 App 端

ECShop App 已安装，但现阶段不具备与本地 Web 端完全相同的业务和数据条件，因此放在项目最后，作为 Appium 自动化扩展，不阻塞 Web 主项目。

## 3. 项目角色

| 角色 | 主要权限与业务 |
| --- | --- |
| 游客 | 浏览、分类查询、商品搜索、查看详情、注册、登录 |
| 注册用户 | 用户中心、购物车、收货信息、下单、订单查询、退出登录 |
| 管理员 | 商品、分类、会员、订单和系统配置管理 |

## 4. 测试目标

1. 验证 ECShop 核心电商业务是否正确、稳定。
2. 验证游客、注册用户和管理员之间的权限边界。
3. 验证 Web 页面、接口响应和 MySQL 数据的一致性。
4. 使用接口自动化和 Web UI 自动化提高核心回归效率。
5. 使用 JMeter 和 LoadRunner评估核心业务的性能表现。
6. 使用 Jenkins 定时或按需执行自动化测试并发布 Allure 报告。
7. 输出完整测试文档、缺陷记录、质量结论和项目复盘。

## 5. 核心质量关注点

- 注册信息唯一性和数据有效性
- 登录认证、会话和退出状态
- 商品价格、库存和上下架状态
- 购物车商品、数量和金额计算
- 订单重复提交与数据一致性
- 普通用户之间的数据隔离
- 后台操作对前台数据的影响
- 接口业务成功与 HTTP 成功的区别
- 并发登录、商品查询和下单的性能
- 自动化脚本的稳定性、可维护性和报告可读性

## 6. 暂不纳入主线的内容

- 真实在线支付
- 真实短信、微信、QQ 等第三方服务
- 大规模浏览器兼容性实验室
- 生产环境破坏性压力测试
- App UI 自动化，待主项目完成后开展

## 7. 最终项目产出

- 项目介绍和测试计划
- 需求分析和业务流程
- 功能测试用例与执行记录
- Bug 清单与回归记录
- Postman 接口集合和接口测试报告
- MySQL 数据校验 SQL
- pytest 接口自动化框架
- Selenium Web UI 自动化框架
- JMeter 与 LoadRunner 性能报告
- Jenkins 持续集成记录和 Allure 报告
- 测试总结报告
- 简历项目经历和面试讲解稿
