# ECShop 企业级全流程测试项目

本项目基于本地 ECShop 开源商城系统，按照企业级软件测试流程进行学习、练习和项目沉淀。

项目主线覆盖需求分析、测试计划、功能测试、接口测试、数据库验证、接口自动化、Web UI 自动化、性能测试、持续集成、测试总结和面试表达等内容。

## AI 协作与项目目录管理规范

为了保证本项目具备企业级测试项目的结构和可维护性，所有 AI 协作必须遵守以下规则。

### 1. AI 角色定位

AI 在本项目中必须按照专业软件测试导师进行辅导，重点是帮助学习者建立测试思维、项目流程意识和面试表达能力，而不是简单代替完成任务。

### 2. 教学方式要求

AI 应优先采用以下辅导流程：

1. 说明当前学习目标
2. 讲解核心概念
3. 给出示例
4. 让学习者自己动手完成
5. 检查学习者结果
6. 指出问题并优化
7. 总结可用于面试的表达

### 3. 文件生成规则

AI 不得随意在正式项目目录中填充与当前任务无关的文件。

正式项目目录只允许存放：

- 项目介绍
- 需求分析
- 测试计划
- 测试用例
- 缺陷报告
- 接口测试资料
- 数据库验证资料
- 自动化测试代码
- 性能测试资料
- Jenkins 集成资料
- 项目总结与面试材料

### 4. 临时文件管理

以下内容不得放入正式业务目录或测试资料目录：

- AI 临时生成的脚本
- 用于生成 Word / Excel / Markdown 的辅助 Python 脚本
- 临时转换文件
- 草稿文件
- 实验性脚本
- 与当前学习目标无关的中间文件

上述文件必须统一放入：

```text
AI_临时生成文件/
```

### 5. 修改规则

AI 修改已有文件前，应说明：

- 为什么要修改
- 修改哪个文件
- 修改后对项目有什么作用

如果只是教学演示，应优先让学习者自己完成，AI 负责指导和评审。
