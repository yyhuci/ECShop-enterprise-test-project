# ECShop 系统模块清单

| 一级模块 | 子功能 | 当前确认情况 | 后续关注点 |
| --- | --- | --- | --- |
| 首页 | 导航、分类、推荐商品 | 已确认入口 | 商品展示与后台配置是否一致 |

| 登录 | 账号密码、会话 | 已确认入口 | 正确/错误登录、Cookie、锁定策略 |

| 商品分类 | 分类商品列表 | 已确认入口 | 分类准确性、分页、空分类 |
| 商品搜索 | 关键词搜索 | 已确认页面 | 精确/模糊/空值/特殊字符 |
| 商品详情 | 名称、价格、库存、购买 | 已确认入口 | 数据一致性、上下架、库存 |

| 退出登录 | 清除会话 | 待实际操作 | 退出后页面和接口权限 |

## 1.1 已确认的注册字段

| 页面字段 | 请求参数 | 初步规则 |
| --- | --- | --- |
| 用户名 | `username` | 必填；不少于 3 个字符；只能由字母、数字和下划线组成 |
| 邮箱 | `email` | 必填；必须符合邮箱格式 |
| 密码 | `password` | 必填；不少于 6 个字符 |
| 确认密码 | `confirm_password` | 必填；必须与密码一致 |
| 密码提示问题 | `sel_question` | 是否必填待验证 |
| 密码问题答案 | `passwd_answer` | 与提示问题的关联规则待验证 |
| 用户协议 | `agreement` | 必须勾选 |
| 扩展字段 | `extend_field2` 至 `extend_field5` | 实际含义和校验规则待确认 |

注册请求通过 `user.php` 提交，业务动作参数为：

```text
act=act_register
```

## 1.2 已确认的登录字段

| 页面字段 | 请求参数 | 初步规则 |
| --- | --- | --- |
| 用户名 | `username` | 必填 |
| 密码 | `password` | 必填 |
| 记住登录状态 | `remember` | 可选，勾选值为 `1` |
| 返回地址 | `back_act` | 登录成功后的返回页面 |

登录请求通过 `user.php` 提交，业务动作参数为：

```text
act=act_login
```

## 2. 后台管理模块

后台入口已确认可访问。具体菜单需要登录后按实际页面补充，预计重点包括：

| 模块 | 核心业务 | 测试关注点 |
| --- | --- | --- |
| 商品管理 | 新增、编辑、上下架、库存 | 前后台一致性、必填与边界 |
| 分类管理 | 新增、编辑、层级维护 | 分类层级、删除影响 |
| 会员管理 | 查询、编辑、状态管理 | 权限、数据准确性 |
| 订单管理 | 查询、确认、发货、取消 | 状态流转、金额、库存 |
| 促销管理 | 优惠和促销活动 | 生效时间、叠加规则 |
| 系统设置 | 商店和业务配置 | 配置生效范围与权限 |

## 3. 跨端数据链路

| 操作 | Web 预期 | 数据库预期 | 后台预期 |
| --- | --- | --- | --- |
| 用户注册 | 注册成功并可登录 | 新增用户记录 | 可查询到该会员 |
| 后台新增商品 | 前台可按规则展示 | 新增商品记录 | 商品保存成功 |
| 加入购物车 | 购物车数量和金额变化 | 按系统实现保存数据 | 通常无需后台操作 |
| 提交订单 | 返回订单号 | 新增订单及明细 | 后台可查询订单 |
| 后台发货 | 用户订单状态更新 | 订单状态字段变化 | 发货操作成功 |

## 4. 初步风险

1. ECShop 版本较旧，部分页面和接口可能使用传统表单与 Cookie 会话。
2. 第三方支付、短信和社交登录可能无法在本地完整验证。
3. 测试数据可能互相污染，需要建立账号、商品和订单命名规则。
4. 性能测试必须先保证参数化账号和业务断言正确。
5. 本地单机性能结果用于学习和趋势分析，不能直接代表生产容量。
