# 核心业务流程

## 1. 游客浏览流程

```text
进入商城首页
→ 查看商品分类
→ 进入分类列表
→ 查看商品详情
→ 查看价格、库存、积分和评论
```

当前状态：页面入口和商品详情字段已确认。

## 2. 用户注册流程

```text
进入注册页面
→ 输入用户名
→ 输入邮箱
→ 输入密码和确认密码
→ 填写可选扩展信息
→ 选择密码提示问题并填写答案
→ 接受用户协议
→ 提交注册
→ 校验页面结果
→ 校验数据库用户记录
→ 校验后台会员列表
```

执行注册会新增系统数据，因此使用专用测试数据并记录清理方案。

## 3. 用户登录流程

```text
进入登录页面
→ 输入用户名和密码
→ 选择是否记住登录状态
→ 提交登录
→ 进入用户中心或返回原页面
→ 检查 Cookie / Session
→ 退出登录
→ 验证受限页面不可继续访问
```

## 4. 购物车流程

```text
进入商品详情
→ 设置购买数量
→ 加入购物车
→ 查看购物车商品
→ 修改数量
→ 校验单价和小计
→ 删除或继续结算
```

重点检查：

- 促销价是否作为结算单价
- 数量变化后小计是否正确
- 超过库存、0、负数、非数字如何处理
- 重复加入同一商品的行为

## 5. 下单与后台处理流程

```text
登录用户进入购物车
→ 进入结算
→ 填写或选择收货信息
→ 选择配送和支付方式
→ 确认商品和金额
→ 提交订单
→ 获取订单号
→ 查询数据库订单与明细
→ 后台查询并处理订单
→ 前台用户中心检查订单状态
```

此流程会生成订单数据，正式执行前需要确定测试账号、商品和清理策略。

## 6. 后续测试层次

同一业务流程将从不同层次重复验证：

| 层次 | 验证内容 |
| --- | --- |
| 功能测试 | 页面交互和业务结果 |
| Postman 接口测试 | 请求参数、Cookie、响应和业务断言 |
| 数据库测试 | 用户、商品、购物车、订单数据 |
| 接口自动化 | 核心流程稳定回归 |
| Web UI 自动化 | 用户关键操作回归 |
| 性能测试 | 登录、搜索、商品查询和下单容量 |
